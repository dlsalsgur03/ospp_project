## 1. 프로젝트 개요
교내 BLE 센서 데이터를 모바일로 수집하고 API로 서버에 전송하는 시스템을 완성했습니다.
데이터 위에서 무엇을 할지 다양한 아이디어가 나왔으나, 우리가 수집하는 데이터로는 분석이나 ML을 해보기에는 부족하다는 생각이 들었습니다. 
그에 따라 충북대학교 학생들이 자발적으로 데이터를 수집하도록 유도하는 게임을 만들기로 하였습니다.

## 2. 핵심 기능
📊 BLE 센서 데이터 수집  
라즈베리파이의 온/습도·eCO2·공기질·RSSI 데이터를 BLE 광고 패킷으로 수신. 사용자는 지도에서 가까운 센서를 클릭하면 된다.  

💾 서버 전송 및 저장  
전송 버튼으로 측정값을 HTTP로 업로드. 센서 ID·시간·위치와 함께 저장  

🕙 정각 활성화 및 중복 방지  
전송 버튼은 매 정각마다 재활성화. 한 번 전송한 센서는 다음 정각까지 비활성화. 무한 클릭이 아닌 실제 이동을 유도한다.
  
👾 랜덤 캐릭터 출몰  
센서마다 충북대 캐릭터가 랜덤 등장.  "도서관에 우왕이!" 같은 알림으로 발길을 끈다.
  
🏆 전송 횟수 기반 랭킹  
서버에 데이터를 많이 전송한 사용자가 높은 순위. 학과·단과대 단위 랭킹으로 자연스러운 경쟁 유발한다.  

⭐️ 경험치 및 레벨업  
전송 1회당 경험치 적립 → 레벨업
 등수 경쟁과 별개로 "내 성장"을 보여줘 중간 사용자도 이탈하지 않게 한다.  


## 3. 시스템 아키텍처
1. Raspberry Pi Sensors  
-> 온도, 습도, eCO2, TVOC, AQI, RSSI
2. Mobile Application  
-> BLE 스캔, 센서 데이터 수신, 위치/근접 판단, 캐릭터 출몰
3. Server  
-> 수신 API, 저장, 순위, 경험치, 캐릭터 등장 로직
4. Database  
-> 센서, 사용자, 캐릭터, 랭킹(단과대/학과/개인)
5. App Screen / Feedback  
-> 센서 정보, 알림, 기록, 랭킹, 레벨
